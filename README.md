# Kareem Ahmed

I build AI-powered web products. The motion is the product, not the decoration.

Three years in motion design before I moved into engineering, and it still runs
everything I ship: timing, rhythm, and the small details that decide whether an
interface feels intentional or just functional. I care about why a spring beats a
tween, and I can build either.

### What I'm building

**[Nitro Studio](https://nitrostudio.co)**
The first real product I'm shipping, a studio site for a client. The launch page is
live, the full site is in progress. Small on purpose and built around motion, with
numbers to back the feel: Real Experience Score of 100 on live traffic, 64ms
interaction latency, sub-second LCP, zero layout shift.

**[convit](https://github.com/Kareem-AEz/convit)**
Conventional commits by AI, done right. It runs fully local so your code never leaves
your machine, and it pre-analyzes the diff with weighted voting so the message
describes the change instead of saying `fix: update stuff`.

**Ellara**
An AI learning product. In active build, private for now.
